코드 파일에서 UI 용어(st.metric 레이블, 테이블 컬럼, 차트 축, 탭 이름)를 자동 추출하여 GLOSSARY.md Section C와 대조한다.

**인수**: `$ARGUMENTS`가 있으면 해당 파일만 스캔, 없으면 `home.py` + `pages/*.py` 전체 스캔.

---

## Step 1 — 코드 스캔

대상 파일에서 아래 패턴을 추출한다:

| 카테고리 | 추출 패턴 |
|---------|----------|
| **metric** | `st.metric(label="..."` 또는 `st.metric("..."` → 첫 번째 문자열 인수 |
| **탭** | `st.tabs([...])` → 리스트 내 문자열들 |
| **테이블 컬럼** | `column_config = { "컬럼명": ... }` → 딕셔너리 키, `st.column_config.XXX("표시명")` → 첫 인수 |
| **차트 축** | Plotly `title="..."`, `xaxis_title=`, `yaxis_title=` 등 → 문자열 값 |

**추출 규칙**:
- 이모지 접두사(`📈`, `🌐`, `📊` 등)가 붙은 탭/헤더는 이모지 제거 후 순수 텍스트만 추출
- f-string 내 변수 부분(`{var}`)은 제외하고 고정 텍스트만 추출
- `st.subheader`는 스캔 대상에서 **제외** (동적 텍스트 다수)
- 빈 문자열, 순수 숫자, 변수만으로 구성된 값은 제외

## Step 2 — GLOSSARY.md 대조

1. `GLOSSARY.md`를 읽고 Section C(페이지별 용어 카탈로그)에서 각 페이지의 등록된 용어 목록을 파싱
2. Step 1에서 추출한 용어와 비교하여 **누락 항목** 목록 생성
3. 역방향도 확인: GLOSSARY.md에는 있지만 코드에서 사라진 용어도 보고

## Step 3 — 보고

누락 항목을 카테고리별로 정리하여 사용자에게 보고:

```
## 누락 항목 보고

### pages/새페이지.py
#### st.metric (2건 누락)
- `새 지표명`
- `또다른 지표`

#### 테이블 컬럼 (1건 누락)
- `새 컬럼명`

### 삭제된 용어 (코드에서 제거됨)
- pages/구페이지.py: `옛날 지표` (metric)
```

- 누락이 없으면 `✅ 동기화 완료 — 모든 UI 용어가 GLOSSARY.md에 등록되어 있습니다.` 출력
- Section A 표기 규칙 위반 사항도 함께 보고 (수식어+지표 공백, 단위 표기 등)

## Step 4 — 반영 (사용자 승인 후)

**반드시 변경 내용 요약을 먼저 보여주고 사용자 승인을 받은 뒤 수정한다.**

1. **Section C**: 해당 페이지 섹션에 누락 용어 행 추가 (기존 테이블 형식 준수)
2. **Section B**: 새로운 지표가 발견되면 계산식·표기 규칙 초안 추가
3. **Section A**: 표기 규칙 위반 코드가 있으면 수정 제안 (코드 수정은 별도 승인)
4. 삭제된 용어는 Section C에서 제거

## 호출 예시

```
/glossary-sync                    # 전체 스캔
/glossary-sync "시장_분석.py"      # 특정 파일만 스캔
```
